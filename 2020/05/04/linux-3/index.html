Linux中常用命令

这几个命令平常不怎么用，但是实际用途还是有的。就先记录一下。

lastb

lastb命令用于显示用户错误的登录列表，此指令可以发现系统的登录异常。单独执行lastb命令，它会读取位于/var/log目录下，名称为btmp的文件，并把该文件内容记录的登入失败的用户名单，全部显示出来。

nc

nc命令是netcat命令的简称。

  • 远程拷贝文件

    server1上的文件拷贝到server2中。

    • server2中运行

      netcat -lp 1234 > 123.txt
      netcat -lp 1234 > 456.txt

    • server1中运行

      nc -w 1 192.168.1.128 1234 < 123.txt
      nc -w 1 192.168.1.128 1234 < 456.txt

scp

scp命令用于在Linux下进行远程拷贝文件的命令，和它类似的命令有cp，不过cp只是在本机进行拷贝不能跨服务器，而且scp传输是加密的。可能会稍微影响一下速度。当你服务器硬盘变为只读read only system时，用scp可以帮你把文件移出来。另外，scp还非常不占资源，不会提高多少系统负荷，在这一点上，rsync就远远不及它了。虽然 rsync比scp会快一点，但当小文件众多的情况下，rsync会导致硬盘I/O非常高，而scp基本不影响系统正常使用。

scp root@192.168.1.128:/home/duyada/scp/123.txt /scp

//复制到本机

scp /scp/456.txt root@192.168.1.128:/home/duyada/scp/

//上传到目标机

cat

一般用来查看文件内容。

查看/etc/profile的前10行内容，应该是：# head -n 10 /etc/profile
查看/etc/profile的最后5行内容，应该是：# tail -n 5 /etc/profile﻿
sed -n ‘5,10p’ filename 这样你就可以只查看文件的第5行到第10行。

将22的文件内容复制到11文件的末尾，cat 22 >> 11

将a.txt文件的内容清除，并且使得文件的大小为0，而不删除文件可以：

cat /dev/null > a.txt

>>的意思是追加的意思

> 的意思是重定向的意思，会覆盖原先的内容

Linux常见命令的不同形式

最近在搞各种版本的Linux，命令的形态不一样，有时候可以用，有时候不能用，为了清楚的区分它们之间的关系，做一些总结。也算一个补充吧。

service、chkconfig和systemctl

总结

service命令：可以启动、停止、重新启动和关闭系统服务，还可以显示所有系统服务的当前状态。

chkconfig命令：是管理系统服务(service)的命令行工具。所谓系统服务(service)，就是随系统启动而启动，随系统关闭而关闭的程序。

systemctl命令：是一个systemd工具，主要负责控制systemd系统和服务管理器。

systemctl命令是系统服务管理器指令，它实际上将 service 和 chkconfig 这两个命令组合到一起。

service

service命令其实是去/etc/init.d目录下，去执行相关程序。

service redis start
service redis stop

chkconfig

chkconfig是管理系统服务(service)的命令行工具。所谓系统服务(service)，就是随系统启动而启动，随系统关闭而关闭的程序。

chkconfig --list [name]
chkconfig --level 2345 服务名称 off  #关闭开机自启动

systemctl

systemctl是systemd的进程管理命令，systemd是Linux系统最新的初始化系统(init)，作用是提高系统的启动速度，尽可能启动较少的进程，尽可能更多进程并发启动。

systemctl start redis
systemctl stop redis
systemctl enable redis  # 开机自启动
systemctl disable redis

iptables、firewall和ufw

总结

iptables是Linux下功能强大的应用层防火墙工具。

firewall是centos7里面新的防火墙管理命令

ufw是Ubuntu下的一个简易的防火墙配置工具。

iptables

Linux系统都存在的防火墙工具

firewall

firewall是centos7里面新的防火墙管理命令，底层还是调用iptables来处理的，主要区别是iptables服务，每次更改都意味着刷新所有旧规则并从/etc/sysconfig/iptables读取所有新规则，firewall可以在运行时更改设置，而不丢失现有连接。

ufw

配置简单

apt和apt-get

总结

apt 可以看作 apt-get 和 apt-cache 命令的子集, 可以为包管理提供必要的命令选项。

apt-get 虽然没被弃用，但作为普通用户，还是应该首先使用 apt。

apt

apt 命令的引入就是为了解决命令过于分散的问题，它包括了 apt-get 命令出现以来使用最广泛的功能选项，以及 apt-cache 和 apt-config 命令中很少用到的功能。
