python 2安装requests模块报错

电脑里有python2和python3两个环境。配置起来容易出问题。

Windows上Python2和3如何兼容

官方的解法是什么？

我们在安装Python3（>=3.3）时，Python的安装包实际上在系统中安装了一个启动器py.exe，默认放置在文件夹C:\Windows\下面。这个启动器允许我们指定使用Python2还是Python3来运行代码（当然前提是你已经成功安装了Python2和Python3）。

如果你有一个Python文件叫 hello.py，那么你可以这样用Python2运行它

py -2 hello.py

类似的，如果你想用Python3运行它，就这样

py -3 hello.py

去掉参数 -2/-3

每次运行都要加入参数-2/-3还是比较麻烦，所以py.exe这个启动器允许你在代码中加入说明，表明这个文件应该是由python2解释运行，还是由python3解释运行。说明的方法是在代码文件的最开始加入一行

#! python2

或者

#! python3

使用pip

当Python2和Python3同时存在于windows上时，它们对应的pip都叫pip.exe，所以不能够直接使用 pip install 命令来安装软件包。而是要使用启动器py.exe来指定pip的版本。命令如下：

py -2 -m pip install XXXX

-2 还是表示使用 Python2，-m pip 表示运行 pip 模块，也就是运行pip命令了。如果是为Python3安装软件，那么命令类似的变成

py -3 -m pip install XXXX

Python2要在代码文件顶部增加一行说明，才能够在代码中使用中文。如果指明使用的Python版本也需要在文件顶部增加一行，那哪一行应该放在第一行呢？

#! python2 需要放在第一行，编码说明可以放在第二行。所以文件开头应该类似于：

#! python2
# coding: utf-8

pip未安装

首先打开 下面的URL

https://bootstrap.pypa.io/get-pip.py

新建get-pip.py 文件张贴上面网址打开的内容

确保你的Python可以正常使用（cmd下面输入Python 可以进入Python交互界面）

进入你存放get-pip.py的目录

python get-pip.py

以上完成，如果还是不能用那就是你的Python的 Scripts 目录没有加入环境变量 。

安装模块遇到的坑

pip install xxxxxx默认使用的是我python2的pip

那么如何安装python3的pip呢？

通过python3 -m pip install cryptography可以正常安装python3模块。

通过python2 -m pip install cryptography可以正常安装python2模块。

报错信息

问题原因

因为用户目录包含中文导致，用户名是中文导致的。

解决方法

在安装目录 Python27\Lib\目录,找到mimetypes.py文件。

在import下添加几行代码。

if sys.getdefaultencoding() != 'gbk': 
 reload(sys) 
 sys.setdefaultencoding('gbk')

完成后保存，在cmd中执行pip install requests
